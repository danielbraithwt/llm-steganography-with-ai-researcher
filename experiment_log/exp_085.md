# Experiment 085: Paraphrase Disruption (Experiment B)

**Cycle:** 85
**Date:** 2026-03-22
**Type:** Phase 2 — Natural channel usage (Experiment B: Paraphrase Disruption)
**Model:** Qwen/Qwen3-4B-Base
**Script:** `scripts/exp_085_paraphrase_disruption.py`

## Motivation

All Phase 2 evidence (exp 078-084) is **probe-based**: linear probes on KV activations show
forward-looking information beyond what the text contains. But probes demonstrate information
presence, not functional relevance. A skeptical reviewer can argue: "The KV cache has extra
correlations, but the model doesn't actually USE them — it reads the text."

**Experiment B provides BEHAVIORAL evidence.** If we paraphrase the CoT text (preserving all
numbers and arithmetic, changing only surface words) and the model's accuracy drops, then the
specific token choices carried hidden information in their K-routing patterns. The model can't
just "read the numbers" — the surrounding non-number tokens also matter.

This is the first non-probing Phase 2 experiment and the only remaining untested direction from
the research spec. It also satisfies the AGENT.md requirement to seek disconfirmation after 3
consecutive confirming experiments (082-084).

## Method

1. Generate 8-shot CoT on GSM8K (200 problems) with Qwen3-4B-Base (greedy decoding)
2. For correct problems, create two modified versions of the CoT body:
   a. **SYNONYM**: Replace ~50% of non-number words with single-word synonyms from a curated
      dictionary (~130 entries covering common CoT verbs, connectives, adjectives)
   b. **RANDOM**: Replace ~50% of non-number words with random unrelated words (positive control)
3. For all three conditions (original, synonym, random):
   - Teacher-force: [8-shot prompt] + [problem] + [CoT body] + "#### "
   - Let model generate from there (max 30 tokens) → parse numeric answer
   - Check correctness against gold answer
4. Compare accuracy across conditions

**Critical design choices:**
- All numbers and arithmetic operators are preserved exactly in both conditions
- Number preservation is verified programmatically
- 8-shot exemplars are NOT paraphrased (only the test problem's CoT)
- The "#### " marker is always present (consistent format across conditions)
- Same model both generates and is tested (self-consistency)

**Statistical analysis:**
- McNemar's test for paired comparison (same problems across conditions)
- Bootstrap 95% CIs for accuracy drop
- Dose-response: correlation between per-problem replacement fraction and correctness
- Token-level and character-level text similarity metrics

## Pre-Registered Predictions

### If hypothesis is TRUE (specific token choices carry hidden info in K-routing):

1. **Original TF accuracy ≥ 90%**: Teacher-forcing the model's own CoT should reproduce
   the original answer with high fidelity. Some loss from tokenization/generation stochasticity.

2. **Synonym accuracy drop ≥ 10%**: Different surface tokens → different K-vectors →
   disrupted routing → accuracy drops. The 10% threshold matches the research spec.

3. **Random accuracy drop > synonym drop**: Random replacement destroys both semantics
   AND routing, while synonyms preserve semantics but disrupt routing. Random should be
   worse.

4. **McNemar p < 0.05 for original vs synonym**: Statistically significant paired difference.

5. **Dose-response within synonym condition**: Problems with higher replacement fraction
   should show lower accuracy (negative correlation between replacement rate and correctness).

6. **Token change fraction for synonym: 15-35%**: Synonym replacement changes individual
   words, so ~15-35% of tokens should differ.

7. **Number preservation: 100%**: All numbers should be preserved exactly.

### If hypothesis is FALSE (model reads semantic content, not specific tokens):

1. **Original TF accuracy ≥ 90%**: Same prediction.

2. **Synonym accuracy drop < 5%**: The model reads the mathematical content (numbers and
   operations), not the specific word tokens. Synonyms preserve semantics → accuracy preserved.

3. **Random accuracy drop >> synonym drop**: Random breaks semantics, synonym doesn't.
   The gap between random and synonym should be large.

4. **McNemar p > 0.10 for original vs synonym**: No significant paired difference.

5. **No dose-response**: Replacement fraction doesn't correlate with accuracy for synonyms.

### This experiment is informative because:

1. **First BEHAVIORAL evidence** in Phase 2 — all prior evidence is probe-based
2. **Independent methodology** — paraphrase disruption tests a completely different aspect
   (token specificity) than probing (information content)
3. **Can disconfirm**: If synonym accuracy ≈ original, the hidden channel exists (probes show it)
   but isn't carried through specific token choices. This would be important information.
4. **Clean controls**: Random replacement validates the setup (should definitely drop accuracy),
   and number preservation ensures math content is identical.
5. **Addresses the "epiphenomenal" concern**: Probes show information; this shows function.

### Critical thresholds:
- **Synonym drop ≥ 10% AND p < 0.05**: STRONG evidence for token-specific hidden info
- **Synonym drop 5-10% AND p < 0.05**: MODERATE evidence
- **Synonym drop < 5% AND p > 0.10**: AGAINST token-specific hypothesis (but compatible
  with hidden channel being non-token-specific)
- **Original TF accuracy < 80%**: EXPERIMENT BROKEN — teacher-forcing setup is faulty

## Execution Notes

- Model: Qwen3-4B-Base, bfloat16, device_map='auto' on RTX 5090
- 200 problems generated, 170 correct (85.0% accuracy — matches prior experiments)
- Synonym dictionary: 200 entries covering verbs, connectives, adjectives, nouns
- Replacement rate: 50% of eligible words (>1 character, non-number)

- Actual random replacement: 17.1 words per problem (50.6% of eligible)
- Token-level change: synonym 24.4%, random 85.7%
- Text similarity (SequenceMatcher): synonym 0.923, random 0.545
- Number preservation: 100% (170/170) for both conditions

- Seed: 42

## Results

### PART A: Full Teacher-Forcing (exp_085)

Teacher-force the COMPLETE CoT + "#### " and let model generate the answer.

| Condition | Accuracy | Drop | McNemar p |
|-----------|----------|------|-----------|
| Original TF | **100.0%** (170/170) | — | — |
| Synonym TF | **100.0%** (170/170) | 0.0% | 1.0 |
| Random TF | **100.0%** (170/170) | 0.0% | 1.0 |

**ALL conditions achieve 100% accuracy.** Even with 85.7% of tokens replaced by random
nonsense words, the model still produces the correct answer.

**Why:** The model is pattern-matching "last number before ####" rather than computing the
answer. Since all numbers and arithmetic are preserved in the paraphrased text, the last
computed number is always present and correct. This makes full TF uninformative for the
hidden channel question — it only tests number extraction, not computation.

### PART B: Partial Teacher-Forcing at 50% (exp_085b)

Teacher-force the FIRST HALF of the CoT, let model GENERATE the remaining computation + answer.

| Condition | Accuracy | Drop | McNemar p |
|-----------|----------|------|-----------|
| Original prefix | **100.0%** (168/168) | — | — |
| Synonym prefix | **99.4%** (167/168) | +0.6% | 1.0000 |
| Random prefix | **94.0%** (158/168) | +6.0% | **0.0020** |

**Partial TF is informative.** The model must GENERATE the remaining computation steps from
the prefix. Original prefix gives 100% (model perfectly continues from its own text). Synonym
prefix gives 99.4% (only 1 problem broken — not significant). Random prefix gives 94.0%
(10 problems broken — significant at p=0.002).

Bootstrap 95% CIs:
- Synonym drop: [+0.0%, +1.8%]
- Random drop: [+3.0%, +10.1%]

### Error Analysis

**Synonym-broken problem (1/168):**
- Problem 62 (gold=25000): Model computed $2,500 instead of $25,000 from the synonym
  prefix. The paraphrased prefix changed the model's computation path for this pension
  calculation. This is a single case — could be noise.

**Random-broken problems (10/168):**
- Problem 15: Model picked different computation path (computed 125-96=29 instead of the
  correct final answer of 125)
- Problem 45: Model computed different article counts
- Problem 77: Model generated correct intermediate "100/2=50" but stopped there instead
  of continuing (the random words confused the model about the problem structure)

The random errors are genuine COMPUTATION errors — the model fails to follow the correct
reasoning chain when the context words are nonsensical. This confirms the positive control
works: random replacement DOES disrupt computation (6%), while synonym paraphrase does NOT
(0.6%).

## Self-Review

### Why This Null Result Is Important

This is the first BEHAVIORAL test of the hidden channel hypothesis. Prior Phase 2 evidence
(probing at number positions, WRRA, position-sweep decodability) demonstrated that the KV
cache carries forward-looking information BEYOND what the text says. But carrying information
is not the same as USING it.

The paraphrase disruption null result constrains the hypothesis:
- The hidden channel EXISTS (probing evidence: STRONG)
- But the hidden channel is NOT carried through non-number token choices
- The model reads the MATHEMATICAL CONTENT (numbers and operations), not the surface words
- Non-number tokens provide context but don't carry essential computation state

### Reconciliation with Probing Evidence

The probing experiments showed V|nums > 0 at ALL positions (not just number positions). How
can V carry info beyond numbers at non-number positions if paraphrasing those positions
doesn't matter?

**Resolution 1: Redundancy.** V at non-number positions carries forward-looking info, but
this info is REDUNDANT with what V at number positions carries. When non-number tokens change,
the model still gets all the computation state it needs from the number-token positions.

**Resolution 2: Problem-level features.** V|nums at early non-number positions reflects
"problem understanding" (difficulty, problem type, answer range) rather than step-by-step
computation state. This information helps predict the answer (explaining the probe signal)
but isn't essential for computation (explaining the paraphrase null result).

**Resolution 3: The hidden channel is real but non-essential.** The model CAN read the text
directly (as the paraphrase result shows). The hidden channel provides supplementary
computation, but the text pathway alone is sufficient. This is consistent with the "lossy
projection" hypothesis — the text is lossy but NOT useless.

### Confounds and Limitations

1. **Low synonym replacement rate**: Only 9.0% of eligible words were actually replaced by
   synonyms (dictionary coverage is ~20% of CoT vocabulary). A more comprehensive paraphrase
   (using LLM-based rewriting) might produce stronger disruption. However, the random condition
   (50.6% replacement) also produced minimal disruption in full TF and only 6% in partial TF,
   suggesting even stronger paraphrase would have limited effect.

2. **GSM8K task specificity**: In GSM8K, the computation is fully determined by the numbers.
   Non-number words provide context (what's being counted, who does what) but don't affect the
   arithmetic. Tasks where non-number context matters more (e.g., logical reasoning, reading
   comprehension) might show paraphrase disruption.

3. **Distributional shift confound in partial TF**: The synonym prefix is slightly out of
   distribution for the model (it generated different words). The 1/168 synonym error could
   be distributional shift rather than hidden channel disruption. The 10/168 random errors
   are DEFINITELY distributional shift (random words are severely out of distribution).

4. **Single model**: Tested only on Qwen3-4B-Base. Cross-model replication (especially on
   Phi, which showed strong WRRA) would strengthen the finding. However, the null result
   likely generalizes because the mechanism (reading numbers from text) is universal.

### Evidence Strength: MODERATE (for the null finding)

- Clean methodology with good controls
- Clear gradient: original > synonym > random (consistent with semantic reading)
- Full TF: 100% across all conditions (model extracts answer from numbers)
- Partial TF: 99.4% synonym (not significant), 94.0% random (significant, p=0.002)
- Below the 10% threshold from the research spec for synonym disruption
- Single model (needs replication for STRONG)

### Implications for the Paper

1. **The hidden channel exists but is not the sole computation pathway.** The text provides
   sufficient information for computation. The hidden channel provides supplementary information
   (as shown by probing), but the text pathway alone works.

2. **This is actually CONSISTENT with the "lossy projection" framing.** If text is a lossy
   projection, there should be information in the hidden states beyond the text (probing shows
   this). But the text should also contain ENOUGH information for the computation (paraphrase
   shows this). Both can be true simultaneously.

3. **The hidden channel operates through NUMBER-TOKEN K/V patterns**, not through non-number
   surface words. This refines the hypothesis and is consistent with the Phase 1 finding that
   K-vectors at number positions are most important.

## Next Steps

1. **Cross-model partial TF on Phi**: Replicate the partial TF finding on Phi-3.5-mini
   (which showed the strongest WRRA). If synonym paraphrase also shows ~0% drop on Phi,
   the null result generalizes.
2. **Number-token perturbation**: Instead of paraphrasing non-number words, change how
   NUMBERS are represented (e.g., "16" → "sixteen", "$18" → "18 dollars"). This tests
   whether number-token-level K/V patterns carry hidden info.
3. **Semantic context disruption**: Use a task where non-number context matters for
   computation (e.g., logical reasoning with "if/then" conditions). GSM8K arithmetic is
   fully number-determined, so non-number paraphrase can't affect it.
4. **Position-sweep replication on Phi**: Still the highest-priority item for cross-model
   evidence strengthening.
