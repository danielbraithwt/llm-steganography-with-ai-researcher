# Experiment 088: Position-Sweep Decodability with Numbers Control — Qwen3-8B-Base

**Cycle:** 88
**Date:** 2026-03-22
**Type:** Phase 2 — Size scaling test (position-sweep decodability, 4B→8B within Qwen family)
**Model:** Qwen/Qwen3-8B-Base
**Script:** `scripts/exp_088_qwen8b_position_sweep_nums.py`

## Motivation

The position-sweep decodability with numbers control is our strongest Phase 2 result:
- **Qwen3-4B-Base** (exp_084): V|nums peak=0.497, p=0.01, 19/20 bins positive
- **Phi-3.5-mini-Instruct** (exp_086): V|nums peak=0.540, p=0.003, 20/20 bins positive
- **Mistral-7B-v0.3** (exp_087): V|nums peak=0.369, p=0.137, 20/20 bins positive (not significant)

**Original plan was Llama-3.1-8B-Instruct, but the model is gated and inaccessible.**
Pivoting to Qwen3-8B-Base, which tests a new and important dimension: **model size scaling**.

Qwen3-8B-Base is ideal because:
1. **Same architecture as Qwen3-4B-Base**: GQA (8 KV heads × 128 dim = 1024 KV dim), 36 layers
2. **2x model size** (8B vs 4B params) — tests whether forward-looking channel scales with capacity
3. **Higher hidden size** (4096 vs 2048) — more internal computation capacity but SAME KV dim
4. **Expected higher GSM8K accuracy** (~80-90%) — provides another strong-signal data point
5. **Directly comparable** to exp_084 (same family, same KV dim, same layers, same probe methodology)

The size scaling question is scientifically important:
- If V|nums is STRONGER on 8B → the channel scales with model capacity (more computation → more forward-looking info)
- If V|nums is SIMILAR → the channel is fully developed at 4B (architecture matters, not size)
- If V|nums is WEAKER → surprising; would need explanation (maybe 8B is less "pre-computation" because it can solve problems in fewer steps)

This also adds a 4th data point to the accuracy × signal gradient:
- Qwen-4B (88%) → V|nums peak 0.497
- Phi (85%) → V|nums peak 0.540
- Mistral (44%) → V|nums peak 0.369
- **Qwen-8B (~85-90%) → ?**

## Method

Same methodology as exp_084/086/087:

1. Generate 8-shot plain-text CoT on GSM8K (250 problems) with Qwen3-8B-Base (greedy decoding)
2. For each correct problem, forward pass to extract V and K at ALL CoT positions at
   L18 (50%) and L27 (75%) — Qwen3-8B has 36 layers (same as Qwen3-4B)
3. At each token position, compute cumulative visible numbers (question + CoT numbers)
4. Bin tokens by relative position (20 bins)
5. At each bin, train probes with GroupKFold (5-fold):
   - V → log(answer): raw V decodability
   - cumNums → log(answer): text-numbers baseline
   - V → log(answer) | cumNums: residualized (genuine forward-looking)
   - K variants
   - Shuffle control
6. Bootstrap (300 iterations) for peak V|nums significance

**Key architectural details (comparison with Qwen3-4B-Base):**
- 36 layers (SAME as 4B)
- 8 KV heads × 128 head_dim = 1024 KV dim (SAME as 4B)
- 32 attention heads (vs 16 for 4B) — more attention heads per KV head (4x vs 2x)
- 4096 hidden size (vs 2048 for 4B) — 2x wider
- Probe layers: L18 (50%) and L27 (75%) — SAME as exp_084

## Pre-Registered Predictions

### If hypothesis is TRUE (V carries forward-looking info, scales with model size):

1. **V_R at 2.5% > 0.30**: V decodability at chain start should be at least as strong as 4B
   (0.44). Slightly lower threshold because 8B might have different answer distributions.

2. **V|nums_R > 0 at ≥3 early bins (0-30%)**: R ≥ 0.05 at ≥3 bins. KEY replication criterion.

3. **V|nums_R peak ≥ 0.40**: Comparable to or exceeding Qwen-4B (0.497). If the channel
   scales with model capacity, it should be at least as strong.

4. **Bootstrap p < 0.05 for peak V|nums**: Statistically significant.

5. **Shuffle R ≈ 0** (±0.05): Validates methodology.

6. **V|nums early mean ≥ 0.15**: Comparable to or exceeding Qwen-4B (~0.23).

7. **nums_R similar to Qwen-4B** (~0.26): Since both models are Base Qwen and expected
   to have similar accuracy, nums_R should be similar.

### If hypothesis is FALSE (forward-looking signal is 4B-specific or doesn't scale):

1. **V_R ≈ nums_R at most positions**: V just encodes visible numbers, even at 8B.
2. **V|nums_R ≈ 0 (±0.05) everywhere**: No forward-looking signal.
3. **Bootstrap p > 0.10**: Not significant.
4. **V|nums_R peak < 0.15**: Much weaker than 4B despite similar accuracy.

### This experiment is informative because:

1. **Size scaling is a new dimension** — Phase 2 has tested 3 different families but never
   2 sizes within the same family
2. **Cleanest possible comparison**: Same architecture, KV dim, layers, family. Only the
   hidden size (computation capacity) differs. Any difference in V|nums is attributable to
   model capacity, not architecture.
3. **If signal is comparable → robust across sizes** — strengthens generality
4. **If signal differs → size matters** — informs about the relationship between model
   capacity and forward-looking computation

### Critical thresholds:
- **V|nums_R peak ≥ 0.40 AND p < 0.05**: REPLICATES and matches 4B — size-independent
- **V|nums_R peak ≥ 0.55 AND p < 0.01**: SCALES UP — stronger at larger size
- **V|nums_R peak < 0.15 AND p > 0.10**: FAILS — 4B-specific
- **V|nums_R peak 0.15-0.40 AND p < 0.05**: MODERATE — size matters but signal present

## Execution Notes

- Model: Qwen/Qwen3-8B-Base, bfloat16, device_map='auto' on RTX 5090
- Architecture: GQA (8 KV heads × 128 head_dim = 1024 KV dim), 36 layers, hidden_size=4096
- 250 problems generated, 224 correct (89.6% accuracy — matches Qwen-4B's 88.0%)
- All 224 correct problems extracted, 19,563 vectors per layer
- Probed at layers 18 (50%) and 27 (75%)
- 20 bins, ~850-1100 tokens per bin, all 224 groups in every bin
- GroupKFold (5-fold), RidgeCV with alpha logspace(-2, 6, 50)
- Bootstrap: 300 iterations per layer
- Total runtime: 1000s (16.7 min) — fastest experiment of the series
- Seed: 42
- Note: Original plan was Llama-3.1-8B-Instruct but model is gated and inaccessible.
  Pivoted to Qwen3-8B-Base (size scaling test).

## Self-Review

### What This Replication Establishes

1. **Size-independent**: The forward-looking channel exists at both 4B and 8B scale within
   the Qwen architecture. Peak V|nums is nearly identical (0.488 vs 0.497).
2. **Architecture-dependent, not size-dependent**: The similarity in signal strength despite
   2x model size suggests the channel is an architectural feature (GQA, digital encoding)
   not a capacity-driven feature.
3. **Layer depth matters more than model size**: L27 (75%) is STRONGER than L18 (50%) on
   8B — peak shifts earlier (bin 15 vs bin 18). Deeper layers decode the answer earlier.

### Interesting Findings

1. **nums_R asymmetry**: Despite similar accuracy (88% vs 90%), the 8B model's answers are
   much more predictable from input numbers (nums_R 0.42 vs 0.26). This could mean the 8B
   model "thinks more linearly" — its computations are more transparently related to inputs.
   Or it could be that the 8B model generates CoT with more explicit intermediate numbers.

2. **Early signal is weaker on 8B**: V|nums at 2.5% is 0.218 (8B) vs 0.357 (4B). The 8B
   model may encode answer information more gradually across the chain, while the 4B model
   frontloads it more. This is consistent with the 8B model having "room" for more gradual
   computation.

3. **L27 significance (p=0.013) vs L18 marginal (p=0.083)**: The deeper layer carries more
   reliable forward-looking information. This is consistent with computation accumulating
   through layers — the answer signal gets refined and strengthened at deeper representations.

### Confounds and Limitations

1. **Same family**: Qwen-4B and Qwen-8B are from the same family. This doesn't add a NEW
   family to the cross-model count. The replication shows size-independence within a family
   but doesn't strengthen the cross-family argument.

2. **nums_R asymmetry is unexplained**: Why does the 8B model have much higher nums_R?
   This might reflect different answer distributions, different CoT structures, or
   different regularities in the 8B model's reasoning patterns. Without further analysis,
   we can't definitively rule out that V|nums is weaker on 8B because nums_R absorbs more
   variance.

3. **Probe layer selection**: L18 and L27 for 8B (50%, 75%) vs L18 and L27 for 4B (also
   50%, 75%) — same relative positions but the models might have different computational
   profiles across depth.

4. **Same problem set**: Both models tested on first 250 GSM8K problems. Problem overlap
   is near-total (224/250 correct for 8B vs 220/250 for 4B). Not fully independent data.

### Evidence Strength: MODERATE-STRONG

- V|nums positive 20/20 bins on both layers (consistent)
- L27 p=0.013 (significant)
- Peak V|nums 0.488 (comparable to 4B's 0.497)
- Same family (within-family replication, not cross-family)
- Higher nums_R partially explains weaker early signal
- Establishes size-independence as a new finding
