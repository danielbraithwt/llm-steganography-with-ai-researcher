# Experiment 105: Prompt-vs-Chain Correctness Discrimination Control

**Cycle:** 105
**Date:** 2026-03-23
**Type:** Phase 2 — Confound control for correctness classification
**Model:** Qwen/Qwen3-4B-Base (GQA, 8 KV heads, digital encoding)
**Script:** `scripts/exp_105_prompt_vs_chain_correctness.py`

## Motivation

Exp 104 showed V-cache predicts correctness (AUC 0.63-0.80) from bin 0 (first
5% of chain), with V>text gap of +0.09-0.11 at early bins. At 3/4 layers,
V-AUC exceeded 0.60 at bin 0, when the model has generated only ~5-6 tokens.

However, exp 104's self-review identified a critical confound: **problem encoding**.
The V-cache at chain positions includes prompt KV encoding. At bin 0, the V-vectors
are influenced by the question representation built up during the prompt. A probe
might pick up "this is a hard problem → likely incorrect" from question features
rather than "the computation is going well/poorly."

This experiment directly resolves the confound by comparing:
1. **V at the last prompt token** (pure problem encoding — no chain computation)
2. **V at chain positions** (problem encoding + ongoing computation)

In a causal transformer, V at the last prompt token depends ONLY on tokens 0..T_prompt.
It contains the full question representation but ZERO information about the chain.
If V_prompt predicts correctness as well as V_chain, the signal is problem encoding.
If V_chain > V_prompt, the signal reflects computation state that builds during
generation.

## Pre-Registered Predictions

### If hypothesis is TRUE (V-cache encodes computation quality that builds during generation):
1. **AUC_prompt < AUC_chain_bin0 at ≥3/4 layers.** The signal grows from prompt
   to first chain tokens because computation state is being established.
2. **AUC_chain increases across bins 0→10.** As computation progresses, more
   correctness information accumulates in V.
3. **AUC_prompt ≈ 0.50-0.58.** The prompt encodes problem features but not
   computation quality. Some above-chance performance from problem difficulty.
4. **AUC_combined(prompt+chain) > AUC_prompt** at ≥3/4 layers. Chain features
   add unique information beyond what the prompt provides.
5. **The prompt→chain AUC jump should be largest at mid-depth layers (L18, L27)**
   where computation is most active (from exp 089 layer sweep).

### If hypothesis is FALSE (V-cache signal is from problem encoding, not computation):
1. **AUC_prompt ≈ AUC_chain_bin0.** The signal is already present in the prompt;
   chain generation adds nothing.
2. **AUC flat across bins.** No growth during computation — just static problem features.
3. **AUC_prompt ≈ 0.60-0.70.** The prompt encodes question difficulty/type well.
4. **AUC_combined ≈ AUC_prompt.** Chain features are redundant with prompt features.

### Quantitative predictions:
- Sample: ~500 problems, ~60 incorrect (matching exp 104's rates)
- AUC_prompt: 0.52-0.58 if TRUE, 0.60-0.70 if FALSE
- AUC_chain_bin0: 0.60-0.70 (matching exp 104)
- AUC gap (chain_bin0 - prompt): +0.05 to +0.15 if TRUE, <0.03 if FALSE
- AUC growth across bins: visible increase 0→10 if TRUE, flat if FALSE

### This experiment is informative because:
1. **Directly addresses the #1 confound** from exp 104 (problem encoding)
2. **Clean causal separation**: V at prompt position is causally isolated from chain
3. **Same classification pipeline** as exp 104 — only differs in WHICH position
   the features are extracted from
4. **If prompt AUC is high (FALSE prediction), it fundamentally reinterprets
   exp 104**: the V-cache "knows" correctness because it knows the problem, not
   because it knows its own computation state
5. **If prompt AUC is low (TRUE prediction), it strengthens exp 104**: the
   correctness signal genuinely builds during computation

## Method

1. Generate 8-shot plain-text CoT on GSM8K (500+ problems) with Qwen3-4B-Base
2. Forward pass to get KV cache for full sequence (prompt + chain up to ####)
3. Extract V-cache at L9, L18, L27, L35 at:
   a. **Last prompt token** (position T_prompt - 1) — single position V-vector
   b. **First chain token** (position T_prompt) — single position V-vector
   c. **Chain bins 0-19** (same binning as exp 104) — averaged V-vectors
4. For each extraction point, train logistic regression (balanced weights,
   stratified 5-fold CV, PCA-64) to predict correctness
5. Also extract K-cache at same points for K vs V comparison
6. Text-only baseline at each chain bin (same as exp 104)
7. **Concatenation analysis**: [V_prompt || V_chain_b] → correctness vs V_chain_b alone
   Tests whether chain adds unique information beyond prompt
8. Permutation tests at key positions (prompt, bin 0, bin 10, bin 19)

## Execution Notes

- Model: Qwen3-4B-Base, bfloat16, device_map='auto' on RTX 5090
- 450 valid problems (time budget hit at problem 457/600)
- 393 correct (87.3%), 57 incorrect (12.7%)
- KV dim: 1024 (8 heads × 128 head_dim), PCA-reduced to 64 for classification
- Probe layers: L9 (25%), L18 (50%), L27 (75%), L35 (97%)
- Mean chain length: correct=98.8, incorrect=128.1 (32% longer for incorrect)
- Mean prompt length: 1162.7 tokens
- Classifier: Logistic Regression, balanced class weights, C=1.0, 5-fold stratified CV
- 200 permutation tests (completed L9 all 5 positions + L18 prompt + chain0_single)
- Total runtime: 6068s (101.1 min)
- Seed: 42

## Results

### HEADLINE FINDING: PROBLEM ENCODING CONFOUND CONFIRMED

**V_prompt AUC ≥ V_chain_bin0 at ALL 4 layers.** The V-cache at the last prompt
token (before ANY chain generation) predicts correctness as well as or BETTER
than V-cache at early chain positions.

| Layer | V_prompt | V_chain0 (single) | V_bin0 (avg) | V_bin10 | V_bin19 | Gap (bin0−prompt) |
|-------|----------|-------------------|-------------|---------|---------|-------------------|
| L9    | **0.632** | 0.582 | 0.602 | 0.610 | 0.590 | **-0.030** |
| L18   | **0.679** | 0.680 | 0.659 | 0.678 | 0.719 | **-0.019** |
| L27   | **0.729** | 0.649 | 0.620 | 0.638 | 0.755 | **-0.108** |
| L35   | **0.645** | 0.537 | 0.629 | 0.679 | 0.740 | **-0.015** |
| **Mean** | **0.671** | 0.612 | 0.628 | 0.651 | 0.701 | **-0.043** |

The bin-0 correctness signal that exp 104 attributed to "computation quality"
is ENTIRELY explained by problem encoding in the prompt. The model encodes problem
difficulty/type features in the prompt V-cache at processing time, and the
logistic regression probe picks these up.

### FINDING 2: Late-Chain V EXCEEDS Prompt at Deep Layers

While early chain positions carry LESS correctness info than the prompt, late
chain positions (bin 19, 95-100%) EXCEED the prompt at 2/4 layers:

| Layer | V_prompt | V_bin19 | Gap (bin19−prompt) |
|-------|----------|---------|-------------------|
| L9    | 0.632 | 0.590 | -0.042 |
| L18   | 0.679 | 0.719 | **+0.040** |
| L27   | 0.729 | 0.755 | **+0.026** |
| L35   | 0.645 | 0.740 | **+0.095** |

At L35, the late-chain V-cache predicts correctness 0.095 AUC above the prompt.
This represents genuine computation-quality information that builds during
generation — but only at the END of the chain, not the beginning.

### FINDING 3: Concatenation — Chain Adds Unique Info Only at Late Positions

| Layer | Bin | Prompt AUC | Chain AUC | Combined AUC | Unique chain |
|-------|-----|-----------|-----------|-------------|-------------|
| L18 | 0 | 0.679 | 0.659 | 0.678 | -0.001 |
| L18 | 10 | 0.679 | 0.678 | 0.719 | **+0.040** |
| L18 | 19 | 0.679 | 0.719 | 0.725 | **+0.046** |
| L35 | 0 | 0.645 | 0.629 | 0.645 | +0.000 |
| L35 | 10 | 0.645 | 0.679 | 0.711 | **+0.067** |
| L35 | 19 | 0.645 | 0.740 | 0.696 | **+0.051** |

At bins 0-5, chain adds ZERO unique info (combined ≈ prompt). At bins 10+,
chain adds +0.02 to +0.07 AUC — genuine computation-quality information that
is orthogonal to what the prompt provides.

### FINDING 4: K_prompt Predicts Correctness Too

K-cache at the last prompt token also predicts correctness significantly:

| Layer | K_prompt | V_prompt |
|-------|----------|----------|
| L9    | 0.598 | 0.632 |
| L18   | 0.599 | 0.679 |
| L27   | **0.758** | 0.729 |
| L35   | 0.681 | 0.645 |

At L27, K_prompt (0.758) actually exceeds V_prompt (0.729). Both K and V at the
prompt encode problem features that predict correctness.

### FINDING 5: Permutation Tests — Prompt Signal Is Highly Significant

| Layer | Position | AUC | p-value | sig |
|-------|----------|-----|---------|-----|
| L9 | prompt | 0.632 | 0.000 | *** |
| L9 | chain0_single | 0.582 | 0.030 | * |
| L9 | bin0 | 0.602 | 0.020 | * |
| L9 | bin10 | 0.610 | 0.015 | * |
| L9 | bin19 | 0.590 | 0.040 | * |
| L18 | prompt | 0.679 | 0.000 | *** |
| L18 | chain0_single | 0.680 | 0.000 | *** |

The prompt signal is highly significant (p<0.001 at both tested layers). Chain
positions are also significant but weaker (p=0.015-0.040 at L9).

### FINDING 6: Position-Sweep U-Shape

The full position sweep reveals a characteristic pattern:

- V-AUC starts near prompt level at bin 0 (0.60-0.66)
- **Dips BELOW prompt** at bins 4-5 (0.50-0.60) — mid-chain positions encode
  LESS correctness info than the prompt
- **Rises back** toward and above prompt at bins 10-19 (0.62-0.76)

This U-shape suggests: (1) early chain = prompt info dominates, (2) mid-chain =
computation is active but not yet resolved, (3) late chain = computation outcome
accumulates and adds to prompt info.

## Pre-Registered Prediction Assessment

### If hypothesis TRUE (computation quality builds during generation):
1. **AUC_prompt < AUC_chain_bin0 at ≥3/4 layers:** ✗ OPPOSITE — prompt > chain at 4/4
2. **AUC_chain increases bins 0→10:** ✓ PARTIAL — increases at 3/4 layers but starts
   BELOW prompt
3. **AUC_prompt ≈ 0.50-0.58:** ✗ DISCONFIRMED — prompt=0.63-0.73, much higher
4. **AUC_combined > AUC_prompt at ≥3/4:** ✓ PARTIAL — at bin 10+: 3/4 layers; at bin 0: 0/4
5. **Prompt→chain jump largest at mid-depth:** ✗ Cannot assess (no jump at any layer)

Score: 0.5/5

### If hypothesis FALSE (problem encoding):
1. **AUC_prompt ≈ AUC_chain_bin0:** ✓✓ CONFIRMED — prompt EXCEEDS chain (stronger than predicted)
2. **AUC flat across bins:** ✗ NOT flat — U-shaped, with late bins exceeding prompt
3. **AUC_prompt ≈ 0.60-0.70:** ✓ CONFIRMED — 0.63-0.73 (L27=0.73 slightly above range)
4. **AUC_combined ≈ AUC_prompt:** ✓ CONFIRMED at early bins, DISCONFIRMED at late bins

Score: 2.5/4

**FALSE predictions match better (2.5/4 vs 0.5/5).** The correctness signal at
early chain positions is predominantly from problem encoding. However, the late-chain
computation signal was predicted by NEITHER — it's a new finding.

## Self-Review

### Strengths
1. **Causal isolation**: V at prompt position is causally isolated from chain tokens
   in a causal transformer. This is the cleanest possible control.
2. **Same pipeline**: Identical classifier, PCA, and cross-validation as exp 104.
   Only the extraction position changes.
3. **Large sample**: 450 problems, 57 incorrect — matched to exp 104's sample.
4. **Multiple analyses**: Prompt vs chain, concatenation, K comparison, permutation tests.
5. **Pre-registration**: Clear predictions that were mostly disconfirmed by the data.

### What This Establishes
1. **Exp 104's bin-0 correctness signal is a confound.** The "V knows correctness from
   the start" interpretation is wrong — V knows the PROBLEM from the prompt, not the
   computation quality.
2. **The V>text gap from exp 104 is reinterpreted**: V-prompt encodes richer problem
   features (question structure, semantic complexity, multi-step-ness) than the
   numbers-only text baseline. This is expected — V-cache at the prompt captures the
   full question representation, while the text baseline only sees numbers extracted
   from generated tokens.
3. **Late-chain computation signal is real but modest**: At bins 10-19, chain V adds
   +0.02-0.07 AUC beyond prompt. This IS computation quality, but it's much weaker
   than the problem-encoding signal.

### Confounds and Limitations
1. **Prompt length variation**: Different questions have different lengths, so V at
   the "last prompt token" is at different absolute positions. However, the 8-shot
   prefix (~1100 tokens) dominates, so variation is small (~10%).
2. **Single-position vs averaged**: V_prompt is a single position, while chain bins
   average multiple positions. This could make chain estimates more stable. MITIGATED:
   V_chain0_single (also a single position) shows even LOWER AUC than V_prompt.
3. **Incomplete permutation tests**: Only 7/20 tests completed. But prompt significance
   (p<0.001) is strong where tested.
4. **L27 K_prompt=0.758**: The highest AUC in the entire experiment is K at the prompt,
   not V at the chain. This further supports problem encoding as the dominant signal.

### Alternative Explanations
1. **Meta-cognition at prompt time**: The model doesn't just encode problem features —
   it may encode its own PREDICTED difficulty. If the model "knows" at prompt time
   that a problem is hard for it (not just objectively hard), this is a form of
   meta-cognition that goes beyond static problem features. However, this experiment
   cannot distinguish "objective difficulty encoding" from "self-aware difficulty
   prediction" — both produce the same signature at the prompt.
2. **Chain-length confound persists**: Incorrect problems have 30% longer chains.
   Even at the prompt, the model may be encoding features correlated with chain
   length, which then correlates with correctness. This is a variant of "problem
   difficulty" and doesn't change the interpretation.

### Impact on Previous Findings
- **Exp 104 REINTERPRETED**: The "computation quality" interpretation for early bins
  is WEAKENED. The signal is primarily problem encoding, with a smaller late-chain
  computation component.
- **V>text gap REINTERPRETED**: The gap is from V-cache encoding richer features than
  the numbers-only text baseline, not from hidden computation.
- **Exp 101/102 (functional conditioning) UNAFFECTED**: Those experiments tested V|nums
  for answer VALUE prediction, not correctness classification. The V|nums signal is
  residualized against text — so the prompt-encoding confound doesn't apply (the text
  baseline already controls for problem features). The functional conditioning results
  remain valid.
- **Position-sweep (exp 083, 086) UNAFFECTED**: Those probed for answer value, not
  correctness. Different analysis, different confound.

### Evidence Strength: MODERATE-STRONG (CONFOUND CONFIRMATION)

This is strong evidence AGAINST the "computation quality" interpretation of exp 104's
early-bin signal, and moderately supports a nuanced picture where computation-quality
info emerges only at late chain positions (+0.02-0.10 AUC above prompt). The prompt
itself encodes problem features that predict correctness at 0.63-0.73 AUC — this is
a genuine "problem meta-cognition" finding, just not the computation-quality finding
exp 104 claimed.
