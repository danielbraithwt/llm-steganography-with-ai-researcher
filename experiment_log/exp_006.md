# Experiment 006: SNR Cliff Replication on Llama-3.1-8B

**Cycle:** 6
**Date:** 2026-03-14
**Type:** Cross-model replication experiment
**Model:** meta-llama/Llama-3.1-8B-Instruct
**Script:** `scripts/exp_006_llama_snr_cliff.py`

## Motivation

The KV cache SNR cliff (Exp 3 from research_spec) is one of the strongest pieces of evidence for the hypothesis: a sharp transition from 100% to near-0% accuracy within a 3 dB window (~15→12 dB) demonstrates digital-like fragility in the KV cache encoding. This is inconsistent with "text carries the reasoning" (which would predict gradual degradation).

However, this finding is **Qwen-only**. A skeptical reviewer would note that this is the most mechanistic claim in the evidence base yet has no cross-model replication. Additionally, exp_005 found that Llama is dramatically more fragile under random noise than Qwen (5% random noise → 5% accuracy on Llama vs 56% on Qwen), which open question #8 asks about. The SNR sweep directly probes this: if Llama's cliff is at a higher SNR, that explains the fragility.

This experiment addresses:
- Open question #2: Does the SNR cliff replicate on Llama?
- Open question #8: Why is Llama so much more fragile under random noise?

**Limitation:** Using Llama-3.1-8B-Instruct (base unavailable in cache). Instruction tuning may affect noise robustness.

## Method

Replication of Exp 3 methodology on Llama-3.1-8B:

1. Generate 8-shot CoT traces for 30 GSM8K problems, filter to baseline-correct
2. Teacher-force each trace through the model to build the KV cache
3. Inject calibrated Gaussian noise at 14 SNR levels: [0, 5, 8, 10, 11, 12, 13, 14, 15, 16, 17, 18, 20, 25] dB
4. Noise is calibrated per-layer based on L2 norms of reasoning KV entries
5. Measure both:
   - **Answer accuracy:** Generate freely after the trace with noised KV cache
   - **Token accuracy:** How many next-token predictions match clean trace under noise

Key difference from Exp 3: finer-grained sampling in the expected cliff region (11-18 dB) to precisely locate the transition.

## Pre-Registered Predictions

### If the SNR cliff is architecture-general (replicates on Llama):
- Llama shows a sharp accuracy transition (>80% accuracy drop within ≤4 dB window)
- Based on Llama's fragility in exp_005, predict the cliff is at **17-19 dB** (3-5 dB higher than Qwen's ~14 dB)
- The cliff shape should be similar: S-curve transition, not gradual linear decay
- Token accuracy should degrade more gradually than answer accuracy (same pattern as Qwen)

### If the cliff does NOT replicate:
- Accuracy degrades gradually/linearly with decreasing SNR (no sharp transition)
- No qualitative difference between answer accuracy and token accuracy degradation curves
- This would weaken the "digital encoding" interpretation

### Specific quantitative predictions:
- At SNR 20 dB: answer accuracy ≥ 90% (Qwen was 100%)
- At SNR 15 dB: answer accuracy ≥ 50% (Qwen was 100%)
- At SNR 10 dB: answer accuracy ≤ 20% (Qwen was 0%)
- At the cliff center: token accuracy > answer accuracy by ≥15pp (Qwen showed 82% token vs 93% answer at 14 dB)
- Cliff width (90% → <10%): ≤ 5 dB

### This experiment is informative because:
1. Tests the most mechanistic claim (digital-like encoding) on a second architecture
2. Directly compares cliff position and shape across model families
3. Explains Llama's extreme fragility in exp_005 if cliff is at higher SNR
4. Token vs answer accuracy comparison tests whether these degrade at different rates
