# Experiment 008: PGD Adversarial Null Space Replication on Llama-3.1-8B

**Cycle:** 8
**Date:** 2026-03-14
**Type:** Cross-model replication experiment
**Model:** meta-llama/Llama-3.1-8B-Instruct
**Script:** `scripts/exp_008_llama_pgd_null_space.py`

## Motivation

The adversarial null space is the single strongest mechanistic finding in this research (Exp 4): PGD-optimized perturbations at 377x the signal norm can change the model's answer while preserving EVERY intermediate token prediction. This proves the KV cache carries two functionally separable channels.

However, this has ONLY been demonstrated on Qwen3-4B-Base. It has zero cross-model replication. Given that the SNR cliff (Exp 3) turned out to be Qwen-specific (Exp 007), a skeptical reviewer would reasonably ask: "Is the null space also Qwen-specific?"

The researcher explicitly requested this: "cross-model replication of the KV adversarial experiments on Llama."

Given the digital vs distributed encoding difference between Qwen and Llama (Exp 005, 007):
- If Llama's encoding is truly distributed/analog, PGD may have a harder time finding a null space (each position contributes to both text and answer, so perturbation at any position affects both)
- Alternatively, the distributed encoding might provide MORE null space dimensions (more redundancy = more room for perturbation)

**Limitation:** Using Llama-3.1-8B-Instruct (base unavailable). Instruction tuning could affect the null space structure. Also, PGD on 8B model is more memory-intensive — we use prompt-only attacks (perturb only prompt KV entries) which are more efficient and had 100% success on Qwen.

## Method

Adapted from Exp 4 (kv_distortion_experiment.py) methodology:

1. **Trace generation:** Generate 8-shot CoT traces for 25 GSM8K problems, filter to baseline-correct
2. **Teacher-forcing:** For each valid problem, teacher-force the full trace through the model to build clean KV cache
3. **PGD optimization (prompt-only attack):**
   - Create perturbation tensors for prompt KV entries (requires_grad=True)
   - Objective: maximize change in answer token logits while minimizing change in reasoning token logits
   - Loss = -λ_ans * answer_divergence + λ_text * text_preservation_loss
   - PGD with L2 projection, 50 steps, learning rate 0.01
   - Perturbation scope: ONLY prompt KV entries (positions 0 to prompt_len)
4. **Evaluation:** After PGD, generate freely from perturbed cache and check:
   - Does the generated text match the original? (text preservation)
   - Does the answer change? (attack success)
5. **Metrics:**
   - Attack success rate (answer changed while text preserved)
   - Perturbation norm relative to signal norm
   - Per-layer perturbation distribution

Key adaptations for Llama vs Qwen:
- `attn_implementation="eager"` (required for KV manipulation)
- DynamicCache API via `.layers[i].keys/.values`
- Chunked teacher-forcing (128 tokens) to avoid OOM with eager attention
- GQA: 8 KV heads (vs Qwen's multi-head) — perturbation affects multiple query heads per KV head
- Prompt-only attacks only (reasoning attacks had only 40% success on Qwen and are more complex)

## Pre-Registered Predictions

### If the null space replicates on Llama (hypothesis supported):
- Prompt-only PGD attack succeeds on ≥60% of problems (Qwen was 100% on 6 attacks)
- Adversarial perturbation norm is ≥10x signal norm (Qwen was 377x)
- Text preservation: ≥90% of intermediate tokens unchanged
- Answer changes: generated answer differs from clean answer
- Prediction 1a (distributed encoding = more null space): success rate ≥80%, norm ≥100x
- Prediction 1b (distributed encoding = less null space): success rate 30-60%, norm 10-50x

### If the null space does NOT replicate (hypothesis weakened):
- PGD attack success rate ≤20%
- Perturbation norm ≤5x signal norm before text starts degrading
- Cannot change answer without also changing text
- This would suggest the null space is architecture-specific (like the SNR cliff)

### Specific quantitative predictions:
- At 50 PGD steps: optimization converges (loss decreasing)
- Text loss (cross-entropy on reasoning tokens) remains ≤0.5 nats above clean baseline
- Answer logit divergence ≥2.0 nats (meaningful change in answer distribution)
- Perturbation concentrates in late layers (≥layer 20) based on Exp 5's finding

### This experiment is informative because:
1. Tests the core mechanistic claim on a second architecture
2. The digital vs distributed difference makes this a genuine test (not just parameter variation)
3. Prompt-only attacks are the strongest test case (100% on Qwen)
4. If it fails, combined with the SNR cliff failure, this would seriously weaken the generality claim
5. If it succeeds, combined with the noise ablation replication, this would establish the null space as architecture-general

## Execution Notes

- 25 problems attempted, 17 valid (68% baseline accuracy — consistent with exp_005)
- Smoke test: gradient flow check reported NaN gradients due to unstable `logits.sum()` loss (float16 overflow). This was a **false alarm** — the actual PGD with KL divergence loss converged normally with clear optimization trends across all 17 problems.
- All 17 problems ran to completion with 50 PGD steps each
- No OOM errors — 8B model with eager attention + PGD fit in 33.7 GB VRAM
- Total runtime: 199.5 seconds (3.3 min)

| Mean perturbation/signal ratio | **0.8x** |

### Key Observation: The Null Space is "Shallow"

The PGD optimizer converges effectively:
- Text KL decreases from ~600 to ~4 over 50 steps (text preserved)
- Answer KL increases from 0 to ~1700 over 50 steps (answer distribution changed)
- Text-region argmax match recovers from ~10% (step 10) to ~97% (step 40)
- Answer-region argmax match drops to ~0-10% (predictions completely changed)

**But the changed predictions produce GARBAGE, not a valid alternative answer:**

| Problem | Clean Answer | Perturbed Generation |
|---|---|---|
| #1214 | 5 | "Jordan Jordan Jordan Jordan..." (repeated tokens) |
| #57 | 83 | "a a a a a a a..." (repeated filler) |
| #1252 | 70 | "footballfootball football..." (repeated tokens) |
| #618 | 36 | "seats seats seats seats..." (repeated tokens) |

The PGD successfully changes the answer-region predictions while preserving text-region predictions, but the changed predictions don't converge to a **valid alternative answer** — they produce incoherent repetition. This is qualitatively different from Qwen (Exp 4), where PGD redirected the model to produce a different but coherent numerical answer.

### PGD Optimization Dynamics (typical problem)

| Step | Text KL | Answer KL | Text Match | Loss |
|---|---|---|---|---|
| 0 | 0.00 | 0.00 | 100.0% | 0.00 |
| 10 | 619.96 | 197.28 | 6.2% | -366 |
| 20 | 252.01 | 1205.43 | 49.4% | -5775 |
| 30 | 25.18 | 1458.01 | 92.6% | -7265 |
| 40 | 6.69 | 1652.33 | 96.3% | -8255 |
| 50 | 3.85 | 1711.62 | ~97% | -8554 |

The optimizer clearly converges: text KL → 0, answer KL → large, text match → 97%+. The problem is not optimization failure — it's that the ANSWER SPACE on Llama doesn't support valid redirection through prompt-only perturbation.

### Interpretation: Why the Null Space Fails on Llama

1. **Distributed encoding**: Llama's answer information is spread across many positions (confirmed by exp_005 and exp_007). Perturbing only prompt positions cannot coherently redirect the distributed answer computation — it can only disrupt it.

2. **No concentrated answer channel**: On Qwen, the PGD found specific positions/layers where answer information concentrates (rho=0.78 with answer-coupled positions). On Llama, this concentrated structure may not exist at the same scale.

3. **Instruction tuning robustness**: Llama-3.1-8B-Instruct has been heavily tuned to produce specific output formats. The answer generation mechanism may be more robust to KV cache perturbation than Qwen's base model.

4. **Format vs content**: The PGD can change WHAT the model predicts at answer positions (0% answer-region match), but cannot maintain the FORMAT of the answer (no "####" or "answer is" pattern). On Qwen, the format was preserved while the content changed.

## Self-Review

### Confounds
1. **Base vs Instruct model**: Llama-Instruct may have format-preservation mechanisms from instruction tuning that Qwen-Base lacks. This could explain the format collapse.
2. **Model size**: 8B vs 4B — larger models may have more robust answer computation.
3. **GQA vs MHA**: GQA's sharing of KV heads across query heads may create coupling that prevents clean null-space separation.
4. **Answer region definition**: Using last 10 reasoning tokens as "answer region" is arbitrary. A different boundary might give different results.
5. **Optimization approach**: KL divergence loss maximizes distribution change, not argmax change. A targeted attack might succeed where this untargeted approach fails.

### Alternative Explanations
1. **PGD insufficiency**: Maybe 50 steps or 0.05 lr is insufficient. But the optimization converges clearly, suggesting more steps wouldn't help.
2. **Wrong loss function**: Maximizing KL doesn't guarantee argmax change. A loss that specifically targets the correct answer token's probability might be more effective.
3. **Prompt-only limitation**: Maybe reasoning-only attacks or full-sequence attacks would succeed on Llama. Prompt-only might be too constrained.
4. **The null space exists but at larger perturbation scales**: The optimizer converged at 0.8x, but maybe the "true" null space requires 100x+ perturbation.

### Statistical Concerns
- n=17: adequate for showing 0% success rate (if true rate were ≥20%, seeing 0/17 has probability ≤2.3%)
- Wilson 95% CI for 0/17: [0.0%, 18.4%]
- Even if the true rate is near the upper CI bound, it's far below Qwen's 100%

### Evidential Strength: **strong (negative replication)**
The PGD null space does NOT replicate on Llama in the strong sense (valid answer redirection). The partial null space (text preserved, answer disrupted) exists but produces garbage, not valid alternatives. Combined with the SNR cliff non-replication (Exp 007), this significantly weakens the generality claim: the "hidden channel" as a structured, exploitable null space appears Qwen-specific.
