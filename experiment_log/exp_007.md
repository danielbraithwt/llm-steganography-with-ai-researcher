# Experiment 007: SNR Cliff Replication on Llama-3.1-8B (retry of exp_006)

**Cycle:** 7
**Date:** 2026-03-14
**Type:** Cross-model replication experiment (retry)
**Model:** meta-llama/Llama-3.1-8B-Instruct
**Script:** `scripts/exp_007_llama_snr_cliff.py`

## Motivation

Exp_006 attempted this exact experiment but FAILED due to a bug: `'DynamicCache' object is not subscriptable` on every problem at every SNR level. Root cause: the script created new DynamicCache objects without using `attn_implementation="eager"`, causing internal model code to fail when subscripting the cache.

The SNR cliff (Exp 3) is one of the strongest mechanistic claims — a sharp 3 dB transition from 100% to near-0% accuracy demonstrates digital-like fragility in the KV cache encoding. But it's **Qwen-only** and has no cross-model replication. This is the #1 gap a reviewer would target.

Additionally, exp_005 showed Llama is dramatically more fragile under random noise (5% random noise → 5% accuracy on Llama vs 56% on Qwen). An SNR sweep directly probes this: if Llama's cliff is at a higher SNR, that explains the fragility.

This experiment addresses:
- **Open question #2:** Does the SNR cliff replicate on Llama?
- **Open question #8:** Why is Llama so much more fragile under random noise?

**Fixes vs exp_006:**
1. `attn_implementation="eager"` (proven to work in exp_005)
2. Tuple-of-tuples for truncated KV cache (avoids DynamicCache creation bugs)
3. In-place KV cache modification with restore from cloned clean tensors
4. Smoke test before full run

## Method

1. Generate 8-shot CoT traces for 30 GSM8K problems, filter to baseline-correct
2. For each valid problem:
   a. Teacher-force through model once to build clean KV cache
   b. Clone key/value tensors as clean references
   c. For each of 15 SNR levels [0, 5, 8, 10, 12, 13, 14, 15, 16, 17, 18, 19, 20, 22, 25] dB:
      - Restore clean cache from clones
      - Add calibrated Gaussian noise to reasoning KV entries (prompt_len to seq_len)
      - Noise calibrated per-layer: noise_norm = signal_norm * 10^(-SNR_dB/20)
      - Build truncated KV cache (tuple-of-tuples) for lookback
      - Re-process last 20 tokens with noised context → token accuracy
      - Generate freely → answer accuracy

## Pre-Registered Predictions

### If the SNR cliff is architecture-general (replicates on Llama):
- Llama shows a sharp accuracy transition (>80% accuracy drop within ≤5 dB window)
- Based on Llama's extreme fragility in exp_005 (5% random noise → 5% acc), predict the cliff is at **17-20 dB** (3-6 dB higher than Qwen's ~14 dB)
- The cliff shape should be S-curve, not gradual linear decay
- Token accuracy should degrade more gradually than answer accuracy (same pattern as Qwen)

### If the cliff does NOT replicate:
- Accuracy degrades gradually/linearly with decreasing SNR (no sharp transition)
- No qualitative difference between answer and token accuracy degradation curves
- This would weaken the "digital encoding" interpretation significantly

### Specific quantitative predictions:
- At SNR 25 dB: answer accuracy ≥ 95% (noise is negligible)
- At SNR 20 dB: answer accuracy ≥ 80% (Qwen was 100%)
- At SNR 15 dB: answer accuracy 30-80% (Qwen was 100%; Llama predicted lower)
- At SNR 10 dB: answer accuracy ≤ 10% (Qwen was 0%)
- Cliff width (90% → <10%): ≤ 6 dB (Qwen was ~3 dB)
- At the cliff center: token accuracy > answer accuracy by ≥10pp

### This experiment is informative because:
1. Tests the most mechanistic claim (digital-like encoding) on a second architecture
2. Directly explains Llama's extreme fragility observed in exp_005
3. Compares cliff position and shape across model families
4. Token vs answer accuracy comparison probes whether the two degrade at different rates

## Execution Notes

**Cycle 8** (this experiment was designed in cycle 7 but the agent crashed before execution).

Fixes applied in cycle 8 to resolve the persistent DynamicCache bug:
1. **Transformers 5.3.0 API change**: `DynamicCache` no longer has `key_cache`/`value_cache` attributes or `__getitem__`. Access is via `past_kv.layers[i].keys` and `past_kv.layers[i].values`.
2. **Truncated cache**: Use `DynamicCache()` + `.update()` instead of tuple-of-tuples.
3. **Chunked teacher-forcing**: Process sequence in 128-token chunks to avoid OOM from large attention matrices with `attn_implementation="eager"`.
4. **`dtype` not `torch_dtype`**: Parameter name changed in transformers 5.3.0.

All 21 valid problems ran successfully at all 15 SNR levels (zero errors). Total runtime: 184s.

### Key Finding

**The SNR cliff does NOT replicate on Llama.** Llama-3.1-8B shows no accuracy degradation under uniform Gaussian noise until the noise equals the signal (SNR 0 dB). This is a qualitatively different noise robustness profile from Qwen's sharp cliff at 14 dB.

### Reconciliation with Exp_005

This result appears to contradict exp_005, which found Llama MORE fragile than Qwen under noise (5% random noise → 5% accuracy). The resolution:

- **Exp_005**: Destroy N% of SPECIFIC POSITIONS completely (replace with noise) → Llama devastated
- **Exp_007**: Add calibrated noise to ALL positions uniformly → Llama robust

This reveals a fundamental architectural difference:
- **Llama**: Distributes information across many positions (**analog/distributed**). Destroying any small subset is devastating (each position contributes essential info), but uniform noise averages out across the distributed representation.
- **Qwen**: Concentrates information more precisely at specific amplitudes (**digital/concentrated**). Uniform noise at moderate levels destroys the precise encoding, but the concentrated information may be more robust to localized ablation.

## Self-Review

### Confounds
1. **Instruct vs base model**: Llama-3.1-8B-Instruct (base unavailable), while Qwen was base model. Instruction tuning could affect noise robustness.
2. **Model size**: Llama 8B vs Qwen 4B. Larger models may be more robust to noise simply from having more redundant representations.
3. **GQA vs MHA**: Llama uses grouped-query attention (8 KV heads for 32 query heads), while Qwen uses multi-head attention. GQA shares KV entries across query heads, which could provide averaging/robustness.
4. **Chunked teacher-forcing**: The original Qwen experiment may have used single-pass teacher forcing. The chunked approach could subtly affect KV cache state.

### Alternative Explanations
1. **GQA robustness**: The most plausible confound. With 8 KV heads serving 32 query heads, each KV entry affects 4 query heads. Noise in one KV position affects all 4 query heads similarly, but the 4 query heads may provide redundancy. This architectural difference alone could explain the robustness.
2. **Model capacity**: 8B parameters may simply provide more noise tolerance than 4B.

### Statistical Concerns
- n=21 problems: adequate for the binary outcome (100% vs 0%), but would benefit from more data points between 0 and 5 dB to precisely locate the transition.
- No intermediate SNR levels between 0 and 5 dB. Adding 1, 2, 3, 4 dB would better characterize the transition.

### Evidential Strength: **strong (negative replication)**
The finding is clear and unambiguous: Llama shows zero degradation at SNR levels that completely destroy Qwen's accuracy. This is not a marginal effect — it's a qualitative difference. The "digital-like fragility" is architecture-specific, not a general property of transformer KV caches.
