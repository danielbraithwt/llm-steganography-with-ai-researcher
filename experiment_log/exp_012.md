# Experiment 012: AC-Aware vs H2O KV Cache Position Protection

**Cycle:** 12
**Date:** 2026-03-14
**Type:** Causal test / compression comparison
**Models:** Llama-3.1-8B-Instruct (primary), Qwen3-4B-Base (secondary)
**Script:** `scripts/exp_012_ac_vs_h2o_protection.py`

## Motivation

Exp 011 showed that H2O heavy-hitter positions do NOT correspond to answer-coupled (AC) positions (Spearman rho = 0.004 on Qwen, 0.11 on Llama). H2O preferentially retains TC-selective positions (57% TC vs 40% AC at 50% retention) and evicts AC-selective ones. But this was purely correlational — it showed WHAT the mismatch is, not WHETHER it matters.

This experiment provides the **causal test**: if we protect positions using H2O rankings vs AC rankings and noise the remaining positions, which strategy better preserves answer accuracy?

The logic: At a given noise fraction f, each strategy PROTECTS the top-(1-f) positions by its scoring metric and NOISES the bottom-f positions. Since H2O ranks AC-selective positions low (from exp 011), H2O-based protection will noise some answer-relevant positions. AC-based protection will noise only answer-irrelevant positions. If the AC/TC framework captures causally relevant structure, AC-protection should outperform H2O-protection for answer accuracy.

## Method

For each model:
1. Generate 8-shot CoT traces for 30 GSM8K problems, filter to baseline-correct
2. Forward pass with `output_attentions=True` → compute AC, H2O, TC scores per reasoning position
3. Forward pass with `use_cache=True` → build base KV cache
4. For each noise_fraction in [0.01, 0.03, 0.05, 0.07, 0.10]:
   For each strategy in [AC, H2O, TC, Random]:
     - Select bottom-f positions by strategy for NOISE (least important by strategy's metric)
     - Clone KV cache, apply norm-matched Gaussian noise to selected positions
     - Truncate cache to N-1, re-process last token, generate answer
     - Check if answer matches true answer
5. Also run: baseline (no noise) and full_noise (all reasoning positions noised)
6. Aggregate: accuracy by strategy × noise_fraction

Noise: Gaussian noise matched in L2 norm to original KV values (same as exp_004/005).

## Pre-Registered Predictions

### If AC-aware protection outperforms H2O (hypothesis confirmed):
- AC_protect > H2O_protect at noise_fraction ≥ 0.05 by ≥ 15pp
- The AC advantage should be largest at 5-7% noise (the regime where exp_005 showed
  the biggest strategy effects)
- TC_protect should be intermediate (TC partially correlates with AC since AC positions
  are hubs, from exp_004/005)
- Random_protect should be worst at all noise fractions

### If H2O and AC perform similarly (null result):
- AC_protect ≈ H2O_protect at all noise fractions (< 10pp difference)
- This would suggest the H2O-AC mismatch doesn't matter for answer accuracy
- Would weaken the practical implications of exp 011

### If H2O outperforms AC (hypothesis disconfirmed):
- H2O_protect > AC_protect
- This would suggest cumulative attention captures something AC misses
- Would force revision of the AC/TC framework

### Specific quantitative predictions (Llama, primary model):

From exp_005 calibration:
- At 5% random noise → ~5% accuracy
- At 5% SelAC noise (top-5% most AC-selective destroyed) → 19% accuracy
- At 5% SelTC noise (top-5% most TC-selective destroyed) → 43% accuracy

My experiment INVERTS this (protect top, noise bottom):
- AC_protect at 5% noise: noises bottom-5% by AC → least answer-relevant → accuracy ≈ 30-50%
  (similar to SelTC noise since both avoid answer-relevant positions)
- H2O_protect at 5% noise: noises bottom-5% by H2O → includes high-AC positions (exp 011) → accuracy ≈ 10-25%
  (worse because some answer-relevant positions are noised)
- Random at 5% noise → ~5% accuracy (from exp_005)
- TC_protect at 5% noise → accuracy ≈ 15-30% (between AC and Random)

Predicted ordering at 5% noise: AC > TC > H2O > Random

At 1% noise: All strategies should give > 50% accuracy (mild noise)
At 10% noise: All strategies should give < 20% accuracy (severe noise)

### Noise set composition predictions:
- AC_protect noises positions with LOW AC scores → mean_AC_of_noised should be lowest
- H2O_protect noises positions with LOW H2O but HIGH AC scores → mean_AC_of_noised should be highest
- This provides the mechanistic link: H2O-protected noise sets contain more answer-relevant positions

### This experiment is informative because:
1. Tests the CAUSAL consequence of exp 011's correlational finding
2. Directly compares AC-aware vs H2O-based KV compression for answer accuracy
3. If confirmed, establishes practical utility of the AC/TC framework for compression
4. The noise set composition analysis reveals WHY strategies differ
5. Cross-model comparison tests generality
