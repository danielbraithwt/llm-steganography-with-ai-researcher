# Experiment 074: Paraphrase Disruption — Token Replacement Test

**Cycle:** 74
**Date:** 2026-03-21
**Type:** Phase 2 — Natural channel usage (Experiment B)
**Model:** Qwen/Qwen3-4B-Base
**Script:** `scripts/exp_074_paraphrase_disruption.py`

## Motivation

All Phase 2 evidence so far is **probing-based** (decodability from KV cache activations).
A skeptical reviewer can dismiss this: "Probing just shows the KV cache processes information
through normal attention — that's expected, not evidence of a 'hidden channel'."

Experiment B provides a fundamentally different type of evidence: **behavioral**. If changing
the surface tokens of CoT (while preserving all numbers and mathematical operations) degrades
the model's answer accuracy, this demonstrates that the specific token choices carry information
beyond their semantic content — information that is lost when tokens change.

This is the ONLY untried Phase 2 direction. Literature warning: Barez 2025 found no paraphrase
accuracy gap on instruction-tuned models. However, our primary model is Qwen3-4B-BASE (not
instruction-tuned), and we are doing token-level replacement (not semantic paraphrasing).

## Method

1. Generate 8-shot CoT on GSM8K (~200 problems) using Qwen3-4B-Base (greedy decoding)
2. For each correctly-solved problem:
   a. Split generated text at "####" marker → CoT tokens + answer trigger
   b. Classify each CoT token as "numeric/structural" (preserve) vs "replaceable" (English words)
   c. Create **synonym** version: replace each replaceable token with its closest embedding
      neighbor (cosine similarity in the model's own embedding space). Top-3 neighbors sampled
      randomly. This preserves approximate semantics while changing exact token identity.
   d. Create **random** version: replace each replaceable token with a random word from the
      vocabulary. This destroys semantics while preserving all numbers.
3. Teacher-force each condition: prompt + modified_CoT + "####" → let model generate answer
4. Compare accuracy across 3 conditions: original, synonym, random
5. McNemar's test for statistical significance of paired accuracy differences

## Pre-Registered Predictions

### If hidden channel hypothesis is TRUE (tokens carry K-routing info):

1. **Original accuracy ~85%** (replicating exp_071/073)
2. **Synonym accuracy drops ≥10pp** (to ~70-75%): different but similar tokens produce
   partially disrupted K-routing patterns. The model can still partially read the math but
   the hidden channel is degraded.
3. **Random accuracy drops ≥30pp** (to ~40-55%): completely different tokens destroy K-routing
   entirely. The model might still extract some answer from preserved numeric tokens but the
   routing is severely disrupted.
4. **McNemar p < 0.05** for original vs synonym (statistically significant drop)
5. **Per-problem agreement: original-synonym < 95%** (some problems switch from correct to wrong)
6. **Dose-response: more replacements → lower accuracy** (monotonic in tertile analysis)

### If hypothesis is FALSE (model reads semantic content):

1. **Original accuracy ~85%**
2. **Synonym accuracy ≥80%** (drop < 5pp): semantically similar tokens convey same mathematical
   content → same answer computation. Small drop from imperfect synonyms.
3. **Random accuracy drops 30-50pp**: semantics destroyed → model can't reason
4. **McNemar p > 0.10** for original vs synonym (no significant difference)
5. **Per-problem agreement: original-synonym > 95%** (almost all problems give same answer)

### This experiment is informative because:

1. **First behavioral (non-probing) Phase 2 evidence** — complements exp_068-073
2. **Directly tests token specificity**: do exact tokens matter, or just their content?
3. **Three levels of disruption**: original (none), synonym (minimal semantic change),
   random (maximal semantic change) — enables dose-response interpretation
4. **Paired design**: same problems across all conditions → McNemar's test
5. **Literature comparison**: Barez 2025 found null on instruction-tuned models — what about base?
6. **Addresses skeptic**: if positive, shows surface tokens carry hidden information that probing
   alone couldn't demonstrate

### Critical threshold:

**10pp drop** from original to synonym is the pre-registered threshold for "evidence FOR
the hidden channel hypothesis." This is the research_spec target (≥10% accuracy drop).

## Self-Review

### Why 100% across ALL conditions?

The **`<<EXPR=RESULT>>` calculator format** is the critical confound. Each arithmetic step is
encoded as `<<EXPR=RESULT>>RESULT` — a completely self-contained numeric structure. For example:

```
Janet sells 16 - 3 - 4 = <<16-3-4=9>>9 duck eggs a day.
She makes 9 * 2 = $<<9*2=18>>18 every day at the farmer's market.
#### 18
```

When we replace English words with random tokens:
```
ated express 16 - 3 - 4 = <<16-3-4=9>>9PEistry backgroundole.
 thirdFIG 9 * 2 = $<<9*2=18>>18 arch internationaloporingceed allowsstream.
#### 18
```

**The `<<EXPR=RESULT>>` expressions are COMPLETELY PRESERVED.** All numbers, operators, and
the structured format remain intact. The model extracts the answer by:

1. Identifying the `<<EXPR=RESULT>>` expressions (preserved)
2. Finding the last computation result (`18` from `<<9*2=18>>`)
3. Outputting that result after `####`

The English words are **completely redundant** in this format. The model doesn't need to
understand "Janet sells duck eggs" to compute the answer — the arithmetic chain is fully
specified by the numeric tokens.

### Is this a bug?

**No.** The sanity check confirms the setup works:
- Original condition: 100% (teacher-forcing correct tokens → correct answer — expected)
- The model IS generating correct, problem-specific answers (18, 3, 70000, etc.)
- Different problems get different correct answers → the model uses the CoT content

The model IS reading the teacher-forced prefix. It's just that the information it needs
(the arithmetic chain) is entirely in the preserved numeric tokens.

### What this means for the hypothesis

**This result is UNINFORMATIVE about the hidden channel hypothesis**, but highly informative
about experiment design:

1. **Not evidence FOR**: Specific non-numeric token choices don't carry hidden information
   that affects accuracy. But this is because the `<<EXPR=RESULT>>` format makes them
   redundant, not because the hypothesis is wrong.

2. **Not evidence AGAINST**: The model extracts answers from numeric tokens. Whether it does
   this through "normal attention to numbers" or through "hidden KV routing at numeric
   positions" is not distinguishable from this experiment.

3. **Key insight**: The `<<EXPR=RESULT>>` format creates a COMPLETE numeric channel that
   makes English text irrelevant. ANY paraphrase experiment using this format will produce
   null results, regardless of the hypothesis.

### Complementarity with probing results (exp_071-073)

The paraphrase null result is actually CONSISTENT with the probing findings:

- **Probing (exp_071-073)**: KV cache encodes computation results at "=" positions
  (V R=0.977). At general CoT positions, the KV-text gap is much smaller (+0.09).
- **Behavioral (exp_074)**: Only numeric positions matter for accuracy. English word
  positions contribute zero information.

Both findings point to the same conclusion: **the computation flows through numeric/arithmetic
positions, not through English word positions.** The hidden channel (if it exists) operates
at computation-specific positions, not at scaffolding text positions.

### Confounds and Limitations

1. **The `<<EXPR=RESULT>>` format is the primary confound.** This format encodes the complete
   arithmetic chain in a structured, self-contained way. English text is redundant by design.
   A proper paraphrase disruption test requires plain-text CoT without this format.

2. **Teacher-forcing correct solutions.** We only tested correctly-solved problems. The model
   always sees a valid arithmetic chain → always extracts the correct answer. Testing on
   incorrect solutions might behave differently.

3. **The "original = 100%" sanity check is trivially true.** When teacher-forcing the same
   tokens the model generated, it produces the same answer. This is expected but means the
   baseline is 100%, not 85%, making drops harder to detect.

4. **We only tested Qwen3-4B-Base.** Llama-3.1-8B-Instruct (which is more text-compliant)
   might behave differently, though the same format confound would apply.

### Evidence Strength: UNINFORMATIVE (design confound)

The result is internally valid (clean methodology, adequate sample size, proper controls),
but the experimental design has a critical confound: the `<<EXPR=RESULT>>` format preserves
complete mathematical information in the numeric tokens, making English text redundant.
The experiment does NOT test what it was intended to test.

**To properly test paraphrase disruption:**
1. Use plain-text CoT exemplars: "Janet sells 16 - 3 - 4 = 9 duck eggs" (no `<<>>`)
2. Or use tasks where English context is essential for mathematical interpretation
   (e.g., "If she BUYS 3 more" vs "If she SELLS 3 more" changes the sign)
3. Or test at the level of reasoning structure (reorder steps, not just words)

## Next Steps

1. **Plain-text paraphrase disruption**: Repeat with exemplars that use "X = Y" notation
   without `<<EXPR=RESULT>>` wrappers. This removes the numeric safety net and makes
   English context more relevant to mathematical interpretation.
2. **Alternative approach**: Instead of paraphrasing, test whether REORDERING CoT steps
   (same content, different order) affects accuracy. Step ordering affects positional KV
   routing even if content is identical.
3. Consider whether the paraphrase direction is fundamentally limited for this task, and
   refocus on strengthening the probing evidence (e.g., hidden-state baseline, paired
   permutation test for V vs expr-embed).
