# Experiment 072: Computation-Position KV Probing — Mistral Cross-Model Replication

**Cycle:** 72
**Date:** 2026-03-21
**Type:** Phase 2 — Natural channel usage (cross-model replication of exp_071)
**Model:** mistralai/Mistral-7B-v0.3
**Script:** `scripts/exp_072_computation_position_mistral.py`

## Motivation

Experiment 071 revealed an enormous KV >> text gap at arithmetic computation positions
on Qwen3-4B-Base (V R=0.975, K R=0.961, text R=0.108). However, this finding was:

1. **Post-hoc** — the computation-position analysis was not the pre-registered primary goal
2. **Single-model** — only tested on Qwen
3. **Weak text baseline** — the token embedding at "=" is identical for all operations,
   making it trivially uninformative

This experiment addresses all three weaknesses:
1. **Pre-registers** the computation-position prediction as the primary analysis
2. **Cross-model replicates** on Mistral-7B-v0.3 (different family, analog encoding)
3. **Adds a stronger text baseline** — average of expression token embeddings (captures
   the arithmetic content the text actually conveys)

Additionally, Mistral's lower accuracy (41.5% vs Qwen's 87%) may produce more
arithmetic errors, enabling WRRA analysis that was blocked on Qwen.

## Method

1. Generate 8-shot CoT on GSM8K (up to 400 problems) using Mistral-7B-v0.3 with
   the same `<<EXPR=RESULT>>` exemplar format
2. Parse all arithmetic operations from generated text
3. Forward pass to extract full KV cache and hidden states
4. At each "=" position, extract:
   - K features: concatenated K-vectors across all heads at "=" (dim=1024)
   - V features: concatenated V-vectors across all heads at "=" (dim=1024)
   - Text embedding (weak baseline): token embedding at "=" position (dim=4096)
   - Expression embedding (strong baseline): mean of token embeddings for all tokens
     in the expression (e.g., for "5*3", average embeddings of "5", "*", "3") (dim=4096)
5. Train ridge regression probes (5-fold CV) to predict log(operation result)
6. Compare K, V, text-at-"=", and expression-mean baselines
7. If enough arithmetic errors exist (n ≥ 10), run WRRA alignment analysis

## Pre-Registered Predictions

### PRIMARY — Computation-position KV vs text (pre-registered replication of exp_071)

**If the computation-position finding REPLICATES on Mistral:**

1. **V-probe R ≥ 0.85** at best layer (Qwen: 0.975; expect slightly lower on analog model)
2. **K-probe R ≥ 0.80** at best layer (Qwen: 0.961; K typically ~0.02 behind V)
3. **Token-embedding text R < 0.20** (Qwen: 0.108; "=" is always same token)
4. **Expression-embedding text R between 0.30–0.70** (captures arithmetic content but
   lacks contextual computation that attention provides)
5. **V-probe >> expression-embedding ≥ 0.20 R gap** (even the stronger baseline
   doesn't match KV, because KV carries attended computation)
6. **V > K at all layers** (replicating exp_071 and exp_069 findings)
7. **Effect increases with depth** (deeper layers encode more computation)
8. **Shuffle control R ≈ 0** (validates probe signal is genuine)

**If the finding is Qwen-specific (DOES NOT replicate):**

1. **K-probe and V-probe R < 0.50** (KV doesn't carry computation info on Mistral)
2. **Expression-embedding R ≈ KV R** (no advantage of KV over text content)
3. This would weaken the Phase 2 evidence significantly

### SECONDARY — Arithmetic error rate and WRRA (exploratory)

1. **Mistral arithmetic error rate: 1-5%** (higher than Qwen's 0.15% due to
   lower overall accuracy, but calculator format may still enforce correct arithmetic)
2. **If error rate ≥ 1% AND n_errors ≥ 10:** V-probe alignment with correct value > 0.55
   (above chance), K-probe alignment ≈ 0.50 (chance)

### This experiment is informative because:

1. **Pre-registered replication** of exp_071's most striking finding
2. **Cross-model** (Mistral vs Qwen, analog vs digital encoding)
3. **Stronger text baseline** addresses the main critique of exp_071
4. **Dual-purpose:** may also yield WRRA data if Mistral makes more arithmetic errors
5. **Directly comparable** to exp_071 (same exemplars, same probe methodology)

## Results

### PRIMARY: Computation-Position Probing (Pre-registered)

| Layer | K-probe R | V-probe R | Expr-embed R | Token-embed R | Shuffle K | Shuffle V |
|-------|-----------|-----------|-------------|---------------|-----------|-----------|
| L8 (28%) | 0.295 | **0.357** | 0.310 | 0.111 | -0.066 | -0.072 |
| L16 (53%) | **0.431** | **0.435** | 0.310 | 0.111 | +0.031 | -0.039 |
| L24 (78%) | 0.390 | 0.348 | 0.310 | 0.111 | -0.070 | -0.066 |
| L31 (100%) | 0.341 | 0.331 | 0.310 | 0.111 | -0.060 | -0.072 |

**Best layer: L16 (53% depth)**
- V-probe: R = 0.435
- K-probe: R = 0.431
- Expression-embed: R = 0.310
- Token-embed: R = 0.111
- V - expr gap: **+0.125**
- V - token gap: **+0.324**
- Shuffle control: R ≈ 0 (validated)

### Key findings:

1. **KV > text REPLICATES** at computation positions. Both K (0.431) and V (0.435) exceed
   both text baselines. Shuffle controls validate the signal is genuine.

2. **Expression-embed is a meaningful baseline** (R=0.310). This captures what the text
   conveys about the arithmetic (the expression tokens "5", "*", "3"). The KV advantage
   over this STRONGER baseline is only +0.125 — much narrower than the +0.85 gap over
   token-embed that exp_071 showed on Qwen.

3. **Effect is MUCH weaker than Qwen** (0.435 vs 0.975). The Qwen effect was almost
   perfect (near-deterministic value encoding in KV); Mistral's is moderate (significant
   but noisy).

4. **V ≈ K** at computation positions (V: 0.435 vs K: 0.431 at best layer). Weak
   V-dominance, unlike the strong V > K pattern on Qwen (0.975 vs 0.961).

5. **Peaks at mid-depth** (L16, 53%) rather than deepening throughout. Qwen peaked at
   97% depth. This is consistent with exp_069's finding that Mistral encodes information
   earlier (L8 was best for aggregate decodability).

### SECONDARY: Arithmetic Error Rate and WRRA

**Error rate: 9.4% (91/968 operations)** — 63x higher than Qwen's 0.15%.
- The `<<EXPR=RESULT>>` format does NOT prevent errors on Mistral
- Only 3 WRRA cases (3.3% of errors; 0.3% of all operations)
- WRRA rate is very low because Mistral's overall accuracy is only 42%

**WRRA alignment analysis (n=91 errors):**

| Layer | K align | V align | Expr align | Text align |
|-------|---------|---------|-----------|-----------|
| L8 | 0.385 | 0.385 | 0.396 | 0.418 |
| L16 | 0.363 | 0.374 | 0.396 | 0.418 |
| L24 | 0.385 | 0.385 | 0.396 | 0.418 |
| L31 | 0.396 | 0.374 | 0.396 | 0.418 |

**All alignment rates BELOW 0.50 (chance).** The KV probes predict the WRITTEN (wrong)
value more often than the correct value. This means:
- At error positions, the KV cache follows the text, not a hidden correct computation
- There is NO evidence of hidden correct computation at error positions on Mistral
- The "smoking gun" WRRA evidence is ABSENT on this model

## Self-Review

### Comparison to pre-registered predictions

1. **V-probe R ≥ 0.85**: NOT CONFIRMED (0.435). Major deviation from prediction.
2. **K-probe R ≥ 0.80**: NOT CONFIRMED (0.431). Major deviation.
3. **Token-embed text R < 0.20**: CONFIRMED (0.111).
4. **Expression-embed R 0.30–0.70**: CONFIRMED (0.310). Important new baseline.
5. **V >> expr-embed ≥ 0.20 gap**: NOT CONFIRMED (+0.125, below 0.20 threshold).
6. **V > K at all layers**: PARTIALLY CONFIRMED (2/4 layers).
7. **Effect increases with depth**: NOT CONFIRMED (peaks at 53%, then decreases).
8. **Shuffle R ≈ 0**: CONFIRMED (-0.07 to +0.03).
9. **Error rate 1-5%**: EXCEEDED (9.4% — more errors than predicted).
10. **WRRA alignment > 0.55**: NOT CONFIRMED (all below 0.42 — below chance).

**Overall: Predictions were miscalibrated.** The computation-position effect is substantially
weaker on Mistral than expected from Qwen extrapolation. The WRRA finding is negative.

### Why the large discrepancy with Qwen?

Several explanations for Qwen R=0.975 vs Mistral R=0.435:

1. **Encoding strategy**: Qwen uses digital K-encoding (sharp cluster structure). Clustered
   representations are MORE linearly decodable — a linear probe can exploit cluster boundaries.
   Mistral's analog encoding distributes information more smoothly → harder for linear probes.

2. **Model accuracy**: Qwen (87%) gets almost all problems right, so its correct-arithmetic
   training set is very clean. Mistral (42%) has many wrong final answers — even at "correct"
   arithmetic positions, the surrounding context may be messy.

3. **KV dim parity**: Both have 1024-dim KV, but Qwen's hidden size is 2560 (ratio 0.40)
   vs Mistral's 4096 (ratio 0.25). KV captures a smaller fraction of Mistral's total capacity.

4. **Architecture differences**: Mistral has sliding window attention (4096), which may
   diffuse computation across positions differently.

### Confounds and limitations

1. **Expression-embed narrows the gap**: The KV advantage over expression embedding (+0.125)
   is genuine but modest. This means MOST of what the KV cache knows at "=" comes from
   attending to the expression tokens — which is the expected mechanism. The remaining +0.125
   is the "hidden computation" signal (transformation beyond aggregation).

2. **WRRA alignment below chance may indicate probe bias**: The RidgeCV probe, trained on
   correct-arithmetic positions, may be systematically biased toward the written value (since
   written = correct for training data). At test time, when written ≠ correct, this bias
   pushes predictions toward the written value. However, this bias would affect all probes
   equally, and text probes show HIGHER alignment (0.418 > 0.385) — meaning KV is actually
   LESS biased toward written values than text.

3. **Small WRRA sample (n=3)**: Only 3 WRRA cases is too few for separate analysis.

### Evidence strength: MODERATE-WEAK

The computation-position finding partially replicates: KV > text is confirmed, shuffle
controls validate, and the expression-embed baseline provides a more stringent comparison.
However:
- The effect is 2.2x weaker than on Qwen
- The expression-embed baseline narrows the gap to +0.125
- WRRA alignment is negative (no hidden correct computation at error positions)
- Predictions were substantially miscalibrated

### What this means for the overall hypothesis

The hidden channel operates DIFFERENTLY on different models:
- **Qwen (digital encoding)**: Near-perfect value encoding in KV at computation positions.
  KV is a highly structured computation substrate. (But WRRA couldn't be tested — n=2 errors.)
- **Mistral (analog encoding)**: Moderate KV advantage over text, but the channel is noisier
  and doesn't encode correct values at error positions. The model may genuinely compute
  through text more than through hidden channels.

This is consistent with Phase 1 finding that Mistral is more "text-resistant" to corruption
(30% compliance vs Qwen's 94%). If Mistral relies more on text for computation, the hidden
channel would carry less independent information → weaker KV probes, no hidden correct
values at error positions.
