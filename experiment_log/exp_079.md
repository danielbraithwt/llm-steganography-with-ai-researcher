# Experiment 079: Problem-Number Residualization — Challenge Experiment

**Cycle:** 79
**Date:** 2026-03-21
**Type:** Phase 2 — Challenge experiment (last 3 experiments confirmed hypothesis)
**Model:** Qwen/Qwen3-4B-Base
**Script:** `scripts/exp_079_number_residualization.py`

## Motivation

**CHALLENGE EXPERIMENT.** Experiments 076, 077, and 078 all confirmed the hypothesis. Per
scientific standards, this experiment MUST be designed to challenge it.

Experiment 078 found V at computation positions predicts the final answer with partial R=0.520
(controlling for local result). This is the strongest Phase 2 finding. But a skeptical reviewer
can raise the **problem-number confound**: V at "=" positions has attended to all prior tokens
via causal attention, including the problem numbers from the question. Since the final answer
is a mathematical function of those numbers, V could predict the final answer simply by
efficiently encoding the input numbers — no hidden "computation" needed.

The partial correlation V→final|local controls for the local computation result but NOT for
problem numbers. This experiment directly attacks this confound by residualizing against
problem numbers and problem embeddings.

## Method

### Phase 1: Generate plain-text CoT (replicate exp_078)
1. Same plain-text exemplars as exp_078
2. Generate CoT for 400 GSM8K problems on Qwen3-4B-Base (greedy decoding)
3. Parse arithmetic expressions, filter to correct problems

### Phase 2: Feature extraction
4. For each correct problem with arithmetic operations, single forward pass
5. At each "=" position, extract V vectors at L27 (77%) and L35 (100%)
6. Extract **problem numbers**: all numeric values from the question text → fixed-length
   log-transformed feature vector (dim=20)
7. Extract **problem embedding**: cumulative mean of token embeddings over the full prompt
   (captures everything the model "knows" before generating CoT) (dim=2560)

### Phase 3: Residualization probes
8. Ten probes at each layer:
   a. V → final: replicate exp_078 (expect R≈0.635)
   b. prob_numbers → final: how predictive are raw problem numbers?
   c. prob_embed → final: how predictive is the full problem context?
   d. local → final: how predictive is the local computation result?
   e. **V → final | prob_numbers**: KEY CHALLENGE — does V carry info beyond numbers?
   f. **V → final | prob_embed**: STRICTEST — does V carry info beyond problem context?
   g. V → final | local: replicate exp_078 partial
   h. V → final | nums + local: combined control
   i. **V → final | embed + local**: strictest combined control
   j. local → final | nums: does local computation add beyond numbers?
   k. Shuffle control

### Phase 4: Bootstrap significance
9. 2000-sample bootstrap CI for V→final|nums partial correlation
10. Bootstrap comparison: V→final vs prob_embed→final

### Phase 5: Figures
- Residualization hierarchy (bar chart of all probes)
- Waterfall chart: how much R survives each control
- Confound decomposition: stacked bar showing R contributions from each source

### Critical issue found during analysis:
Within-problem data leakage in KFold: multiple operations per problem share the same
(prob_embed, final_answer) pair. KFold randomly splits operations, so "training twins" leak

## Results

### CRITICAL METHODOLOGICAL FINDING: Within-Problem Data Leakage

The standard KFold cross-validation used in exp_078 (and initially here) suffers from
within-problem data leakage. With ~4.6 operations per problem, KFold randomly assigns
operations from the same problem to train and test folds. Since all operations within
a problem share the same (prob_embed, final_answer) pair, the probe trivially memorizes
answers from training twins and "predicts" them for test twins.

**Impact:** prob_embed → final is inflated from R=0.344 (GroupKFold) to R=0.977 (KFold).
This in turn deflates V → final | embed from R=0.22 to R=0.04.

**GroupKFold fixes this** by ensuring all operations from the same problem stay in the
same fold. The GroupKFold results are the ground truth.

### PRIMARY RESULTS: KFold vs GroupKFold Comparison

| Probe | KFold L27 | GroupKFold L27 | KFold L35 | GroupKFold L35 |
|-------|-----------|---------------|-----------|---------------|
| V → final | 0.633 | **0.487** | 0.650 | **0.476** |
| nums → final | 0.480 | **0.153** | 0.480 | **0.153** |
| embed → final | 0.977 | **0.344** | 0.977 | **0.344** |
| local → final | 0.481 | **0.464** | 0.481 | **0.464** |
| V → final \| nums | 0.531 | **0.242** | 0.549 | **0.235** |
| V → final \| embed | 0.040 | **0.221** | 0.048 | **0.140** |
| V → final \| local | 0.512 | **0.299** | 0.540 | **0.285** |
| V → final \| n+l | 0.463 | **0.191** | 0.489 | **0.176** |
| V → final \| e+l | -0.001 | **0.215** | 0.001 | **0.134** |
| Shuffle | -0.013 | **-0.017** | -0.051 | **0.032** |

### KEY FINDING 1: V's Forward-Looking Signal SURVIVES Problem-Number Control

**GroupKFold results (ground truth):**
- V → final | nums R = 0.242 (L27), 0.235 (L35) — **CHALLENGE SURVIVED**
- V carries substantial information about the final answer beyond what raw problem
  numbers predict
- Bootstrap CI (KFold): [0.490, 0.571] at L35, p=0.0000 — highly significant
  (GroupKFold bootstrap not computed, but effect is clearly positive)

### KEY FINDING 2: V's Signal PARTIALLY SURVIVES Problem-Embedding Control

**GroupKFold results:**
- V → final | embed R = 0.221 (L27), 0.140 (L35) — **PARTIAL SURVIVAL**
- After controlling for the full problem context (2560-dim prompt embedding), V at
  computation positions still predicts the final answer with R=0.14-0.22
- This residual can only come from V encoding computation performed DURING CoT generation
  (information not available before generation began)
- **This is the key finding: KFold showed R≈0.04 (appeared null), GroupKFold reveals R≈0.18**

### KEY FINDING 3: Exp_078 Partial Correlation Was Inflated

**Exp_078 claimed V → final | local R = 0.520.** With GroupKFold, V → final | local = 0.285-0.299.

The exp_078 partial correlation was inflated by ~0.22-0.24 R units due to within-problem
data leakage. The TRUE forward-looking signal after controlling for local result is
R ≈ 0.29 (L27), not R ≈ 0.52.

Still clearly positive and meaningful, but the effect size was overstated by ~1.8x.

### KEY FINDING 4: V Survives Even the Strictest Combined Control

**GroupKFold: V → final | embed + local:**
- L27: R = 0.215
- L35: R = 0.134

After controlling for BOTH the full problem context AND the local computation result,
V at computation positions still predicts the final answer with R=0.13-0.22.

**This residual represents computation information accumulated during CoT generation
that goes beyond: (a) what was known from the problem statement, AND (b) the current
computation step.**

### KEY FINDING 5: Problem Embedding is Far Less Predictive Than KFold Suggested

**GroupKFold: embed → final R = 0.344** (vs KFold R = 0.977)

The answer is NOT as predictable from the problem statement as KFold suggested. With
proper problem-level CV, the 2560-dim prompt embedding predicts the answer with only
R=0.344 — leaving substantial room for computation information to add value.

**V → final (R=0.487) exceeds embed → final (R=0.344) by +0.143 R.** V at computation
positions carries MORE answer information than the problem context alone.

### Most Important Finding: The Methodological Correction

This experiment's most valuable contribution is the **discovery of within-problem data
leakage** in operation-level probing with standard KFold CV. This affects:

1. **Exp_078**: V → final | local R=0.520 is inflated → true value ~0.29
2. **This experiment**: embed → final R=0.977 is inflated → true value ~0.34
3. **Any probe at computation positions with multiple operations per problem**

The GroupKFold correction is essential for all future probing experiments.

### What the Corrected Results Mean for the Hypothesis

**MODERATE evidence FOR forward-looking computation:**

With proper GroupKFold CV:
- V → final (R=0.49) > embed → final (R=0.34) by +0.14: V carries more answer info
  than the problem context alone
- V → final | embed (R=0.14-0.22): V carries answer info beyond the problem context
- V → final | e+l (R=0.13-0.22): V carries info beyond BOTH context and local result

The effect sizes are MODERATE (R≈0.15-0.22), not STRONG (as the inflated R≈0.52 suggested).
The signal is real but modest.

### Alternative Explanations

1. **V encodes prior computation steps**: V at position P has attended to all prior "="
   positions. Even if V doesn't carry "hidden computation," it naturally encodes the results
   of prior arithmetic operations via attention. These prior results are predictive of the
   final answer (they're part of the computation chain). This is a normal attention mechanism,
   not a "hidden channel" — but it IS computation information beyond the problem statement.

2. **V encodes positional/structural information**: V at "=" might encode information about
   the problem's computational structure (how many steps, what type of operations) that's
   predictive of the answer magnitude. This is less interesting mechanistically.

3. **Insufficient regularization**: With 1024 features and ~341 problems (GroupKFold), the
   V probe might slightly overfit even with ridge regularization, inflating R.

### Confounds and Limitations

1. **No bootstrap significance for GroupKFold results**: Only KFold had bootstrap tests.
   The GroupKFold R=0.22 (L27) might not be significantly > 0 with proper testing.
   (Though the KFold bootstrap showed very tight CIs.)

2. **Single model**: Only Qwen3-4B-Base. Cross-model replication needed.

3. **GSM8K specificity**: Answers are functions of problem numbers. A task where the answer
   is NOT predictable from initial numbers would be more informative.

4. **Exp_078 partial R revised downward**: The R=0.520 finding from exp_078 was a key
   Phase 2 result. It should be revised to R≈0.29 in the evidence ledger.

### Evidence Strength: MODERATE

The challenge experiment reveals:
- V's forward-looking signal is real but smaller than previously claimed (R≈0.2 vs R≈0.5)
- V survives controlling for problem numbers (R=0.24) and problem embeddings (R=0.14-0.22)
- The exp_078 finding needs a methodological correction (R=0.52 → R=0.29)
- Overall: moderate evidence for V carrying computation information during normal generation
